The ABAP RAP folder is included as a design and code artifact for the approval rule master scenario. It should not be described as a deployed or activated RAP application unless ADT activation, Service Binding, and Fiori Preview evidence are added later.

## How to Present It

Recommended wording:

> CAP handles supplier request orchestration and ERP integration behavior, while the ABAP RAP artifacts model an approval rule master that could be implemented in an ABAP Cloud or S/4HANA extension context.

Avoid this wording unless runtime evidence is added:

> Deployed and operated the RAP application in BTP ABAP Environment.
